- Fixed `SmallWebRTCTransport` requiring OpenCV even for audio-only pipelines. `cv2` is now imported lazily, only when a non-RGB video frame actually needs converting, and the `webrtc` extra no longer pulls it in. Video pipelines should install the new `webrtc-video` extra (`pipecat-ai[webrtc-video]`), which now depends on `opencv-python-headless` instead of `opencv-python` to avoid pulling in GUI system libraries.
