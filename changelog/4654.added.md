- Direct functions listed in an `LLMContext`'s tools are now registered with the LLM service automatically — a separate `register_direct_function()` call is no longer required. `LLMContext(tools=...)` and `LLMContext.set_tools()` also accept a plain list of direct functions and/or `FunctionSchema` objects (in addition to a `ToolsSchema`). Added an optional `@direct_function(cancel_on_interruption=..., timeout=...)` decorator to override a direct function's call options. Explicit `register_direct_function()` registration still takes precedence, so advanced cases (advertising a subset of registered tools, advertise-only `FunctionSchema` entries, catch-all handlers, mid-conversation tool swaps) are unchanged.
