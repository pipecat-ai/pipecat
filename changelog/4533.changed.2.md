- When `LLMContextAggregatorPair(realtime_service_mode=True)` and no custom `user_turn_strategies` is provided, the user aggregator now appends `ExternalUserTurnStartStrategy` and `ExternalUserTurnStopStrategy` to the default strategy chain. This makes `on_user_turn_started` / `on_user_turn_stopped` fire from server-emitted `UserStartedSpeakingFrame` / `UserStoppedSpeakingFrame` for realtime services that provide them (OpenAI Realtime, Azure Realtime, Grok, Inworld). For services that don't emit those frames (Gemini Live, AWS Nova Sonic, Ultravox), the appended strategies install harmlessly and never fire — same as before. Custom `user_turn_strategies` opts out of the auto-append; users who want explicit control remain in explicit control.
