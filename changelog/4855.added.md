- Added `TransportParams.audio_out_send_lead_secs`, an opt-in send-ahead jitter buffer for paced output transports (FastAPI WebSocket, WebSocket server/client, and telephony). When set above `0.0`, the transport may push audio that many seconds ahead of real-time playback, so a send stall shorter than the lead (event-loop hiccups, CPU throttling, network jitter) is absorbed without an audible gap and then caught up by a burst.
  - Defaults to `0.0`, which keeps the existing strict just-in-time pacing, so behavior is unchanged unless you opt in.
  - `BotStoppedSpeakingFrame` and graceful end-of-call playout are compensated for the lead so they still align with real playback; interruptions drop the buffered lead immediately (the playout buffer is cleared by telephony serializers).
