- Fixed `GeminiLiveLLMService` failing to generate an initial response when the context has no seeded messages and only a `system_instruction`. `_handle_context` passes `system_instruction=` to `adapter.get_llm_invocation_params`, which makes the Gemini adapter synthesise a placeholder user turn in `messages`; `_create_initial_response` was calling the same adapter without the kwarg, so `messages` came back empty and the `if not messages: return` early-return fired — `send_client_content` was never called and the model never spoke first with `inference_on_context_initialization=True`. Now `_create_initial_response` also passes the system instruction (only when it's actually going to trigger inference), matching the `_handle_context` behavior and letting Gemini generate the initial response driven by the system prompt.
