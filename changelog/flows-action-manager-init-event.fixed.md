- Fixed `ActionManager._ongoing_actions_finished_event` starting **unset** at construction. The event is intended to indicate "no ongoing actions are still running"; with `_ongoing_actions_count = 0` set at init it should be trivially satisfied and the event set. Without the initial `.set()`, an `await event.wait()` invoked before the first `_enqueue_action` call would block indefinitely, breaking the `count == 0 ↔ event set` invariant. Adds a regression test that pins the initial-state check.
