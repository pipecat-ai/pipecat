- Added `MCPClient.tools()`, a simpler way to use MCP tools: `LLMContext(tools=await mcp.tools())` is now all you need — the client connects on first use and the tools register with the LLM automatically. When the session ends, call `close()` from anywhere (e.g. an `on_client_disconnected` handler); wrapping your pipeline in `async with MCPClient(...)` is no longer necessary. Also added `tools_arguments`, to pin argument values you don't want the model to control (they're applied on every call and hidden from the model), and `call_tool()`, to call an MCP tool directly without going through the LLM.
