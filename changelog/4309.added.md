- Added `LatencyWatchdogObserver` for proactive latency monitoring. Unlike the existing reactive metrics (TTFB, processing time) that report after the fact, the watchdog fires callbacks *before* a processor produces its expected frame, enabling use cases like triggering a TTS filler ("Still working on it...") when an LLM is slow, or switching to a fallback service on stalls. The `subscribe()` API supports three modes:
  - per-processor (start/stop on the same processor class, e.g. slow LLM TTFB)
  - spanning (start on one processor, stop on another, e.g. end-to-end user-speech to LLM-done)
  - global (no processor filter, matches any processor in the pipeline)
