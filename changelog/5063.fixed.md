- Fixed `FilterIncompleteUserTurnStrategies` dropping the response after a tool call when the LLM spoke a `✓` acknowledgement first. The `✓` set the per-turn voiced latch (`_user_turn_completion_voiced`), which `_turn_reset` deliberately keeps across responses and only clears on interruption or user-speaking frames. A tool call did not reset it, so the fresh post-tool inference hit the one-completion guard in `_push_turn_text` and every `LLMTextFrame` was dropped: the bot went silent even though the tool ran. The latch is now reset when a `FunctionCallsStartedFrame` is pushed, so the post-tool response speaks. A filler that already voiced `✓` is not truncated (its marker is already `COMPLETE`, so text streams via the complete branch), and the acoustic-detector dedupe protection is unchanged.
