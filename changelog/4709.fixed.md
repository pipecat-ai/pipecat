- Fixed a regression from #4654 where unregistering a tool's handler on its own — via `unregister_function` / `unregister_direct_function`, without changing the advertised tool set — was silently undone, because the next `LLMContextFrame` re-registered the handler from the still-advertised tool (a "zombie"). An explicitly unregistered handler now stays unregistered while its tool remains advertised (so calls hit the missing-handler recovery and the model learns to stop), and is restored only by registering it again, or by re-advertising the tool (removing it from the advertised set, then adding it back).
