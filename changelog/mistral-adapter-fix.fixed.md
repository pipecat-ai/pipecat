- Fixed `MistralLLMAdapter._transform_messages` over-fixing two cases that produced training-data junk (XML `<tool_call>` tags, `IN_PROGRESS`, `[FINAL_ANSWER]` markers) on current Mistral models. (1) Step 1 no longer inserts an empty `{"role": "assistant", "content": " "}` placeholder after a *trailing* tool message — Mistral accepts trailing `"tool"` and generates the next assistant turn from scratch; the placeholder is still inserted mid-conversation when a `"tool"` isn't followed by an `"assistant"`. (2) Step 3 sets `prefix=True` on a trailing assistant only when the content is non-empty; setting `prefix=True` on an empty / whitespace-only / `None` content asked Mistral to invent a continuation from nothing.
