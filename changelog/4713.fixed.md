- Fixed `pipecat init --eval` having no effect on dial-out/SIP telephony scaffolds (`daily_pstn_dialout`, `twilio_daily_sip_dialin`, `twilio_daily_sip_dialout`). These bots build their `DailyTransport` by hand, so the eval transport entry was never generated (while its `WebsocketServerParams` import was, unused). Generated bots now detect eval runs (`EvalRunnerArguments`), build the eval websocket transport via `create_transport`, and skip the dial-out/SIP-only body parsing and event handlers, so the same `bot.py` runs real calls or `pipecat eval` scenarios with `-t eval`.
