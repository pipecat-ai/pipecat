`pipecat create` now accepts an optional target directory. Passing a path — for example `pipecat create .` — scaffolds the project **directly into that directory** instead of nesting it under a `<project-name>/` subfolder (the same convention as `npm create vite@latest .`). The project name defaults to the target directory's basename, and `--name` still overrides it. Omitting the argument keeps the existing behavior, so this is non-breaking.
