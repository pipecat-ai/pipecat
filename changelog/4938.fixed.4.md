The re-prompt timeout was only cancelled on `InterruptionFrame`, which does not fire when the user resumes speaking inside the same open turn. It is now also cancelled on `VADUserStartedSpeakingFrame`, so a user who resumes after a pause is no longer interrupted by a canned "no rush" prompt.
