- Fixed `FlowManager._update_llm_context` broadcasting `LLMUpdateSettingsFrame` (carrying the node's `role_message` as `system_instruction`) without a `service=` scope. `LLMService.process_frame` accepts the frame when `frame.service is None or frame.service == self` — so in a multi-worker pipeline every sibling LLM service picks up the delta and adopts this FlowManager's role. Result: cross-LLM persona / system-instruction hijack during sub-bot handoffs. The FlowManager now sets `service=self._llm` on the frame, restricting the delta to its own LLM.
