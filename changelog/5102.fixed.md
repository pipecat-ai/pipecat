- Fixed streamed-token TTS providers with no word-timestamp support (e.g. `DeepgramFluxTTSService` in its default `TextAggregationMode.TOKEN` configuration) tracking each LLM token as its own spoken unit instead of grouping tokens into sentences, and never sending the RTVI "new segment" bot-output event. Previously the `AggregatedFrameSequencer` only grouped streamed tokens into sentences when the TTS service also supported word timestamps; providers without word timestamps now get the same per-sentence grouping, completing each sentence's slot as soon as it is confirmed (there is no word-timestamp signal to complete it progressively).
