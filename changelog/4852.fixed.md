- Added a `deduplicate_output_messages` setting (default `False`) to `OpenAIResponsesLLMService` and `OpenAIResponsesHttpLLMService`. When enabled, only text from the first `message` output item is forwarded, working around an OpenAI Responses API bug where a single response can carry multiple `message` items with identical content, causing the reply to be spoken/emitted twice. It defaults off to avoid dropping content from responses that legitimately contain multiple distinct `message` items (e.g. a gpt-5.4+ `commentary` message followed by a `final_answer` message).
