- Fixed `WebsocketService` dropping messages and logging spurious "send failed; unable to reconnect" errors during concurrent disconnects. When `_receive_task_handler` and `send_with_retry` both detected a disconnect simultaneously, `_try_reconnect` would return `False` immediately for the second caller ("reconnect attempt aborted: already in progress"), which `send_with_retry` treated as a hard failure — even though the background reconnect was about to succeed. The second caller now waits for the in-progress reconnect to finish via `asyncio.Event` and retries the send on success. Also downgrades the initial send-failure log from `error` to `warning` since a code-1000 server close is a normal teardown.
