- Fixed `SarvamTTSService` (WebSocket) emitting `BotStoppedSpeakingFrame` late. The service never produced a `TTSStoppedFrame` on synthesis completion, so end-of-turn was detected only by the `stop_frame_timeout_s` idle timer, causing `BotStoppedSpeakingFrame` to lag the actual end of audio by up to that timeout (especially for short utterances or a raised `stop_frame_timeout_s`). The service now requests Sarvam's completion event (`send_completion_event`) and emits `TTSStoppedFrame` as soon as the `final` event arrives, so the bot-stopped-speaking event tracks the end of audio. The idle timeout remains as a fallback.
