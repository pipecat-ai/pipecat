- Added `on_user_message_added` event handler on `LLMUserAggregator`, with a new `UserMessageAddedMessage` arg type. It fires when a user message is written to the LLM context, carrying the finalized text. In cascade mode it coincides with `on_user_turn_stopped`; in realtime mode (where the turn-stop event fires before the message is finalized, with `UserTurnStoppedMessage.content=None`) it's the canonical way to subscribe to "context just updated, here's the user text." The assistant message is written at the same time as `on_assistant_turn_stopped` fires in both modes, so existing subscribers there don't need a new event.
