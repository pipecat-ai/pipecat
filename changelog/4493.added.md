- Pipecat pipelines are now agents. The new subagents framework (`pipecat.workers`) turns every `PipelineWorker` into a peer on a shared bus that passes typed messages, dispatches `@job` work, and coordinates with siblings, while existing single-pipeline code keeps running untouched. You're multi-agent by default: drop in a second worker and the bus is already there. `examples/multi-worker/` ships ready-to-run patterns: LLM handoff, parallel debate, sidecar code assistants and hardware controllers, distributed deployments over Redis or PGMQ, point-to-point WebSocket proxies, and UI workers driving a web client over RTVI.
