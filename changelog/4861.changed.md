- `pipecat init` is now the single front door for starting a Pipecat app. After making the project agent-ready (`AGENTS.md` + `CLAUDE.md`), interactive `pipecat init` asks how you want to build: hand off to a coding agent (which also writes the from-scratch developer guide `GETTING_STARTED.md`), or scaffold a runnable bot now — in which case it runs the `pipecat create` wizard in-place, so one directory ends up with both the guide and a runnable bot. The prompt only appears in an interactive terminal, so scripted/CI use is unchanged. `pipecat init quickstart` now scaffolds the canned quickstart bot in-place (previously it redirected to `pipecat create quickstart`) and makes it agent-ready in one step. `pipecat create` is unchanged as the scaffolder that coding agents and automation call directly.

- Scaffolded project READMEs now include a "Building with an AI coding agent" section with Pipecat Context Hub setup, so the project tells you how to give your coding agent live, accurate Pipecat context.
