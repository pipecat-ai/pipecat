- Fixed `ExternalUserTurnCompletionStopStrategy` (and `LLMTurnCompletionUserTurnStopStrategy`) finalizing the user turn while the user was still speaking. The external completion verdict (e.g. an LLM `✓`) can arrive with latency, so the user may have resumed speaking by the time it lands; finalization now re-validates against live VAD and keeps the turn open instead of letting the bot talk over the user. For the LLM-marker path, where the spoken response is emitted from `UserTurnCompletionLLMServiceMixin._push_turn_text` rather than at finalization, the same re-validation is applied there: a `✓` resolving while the user is speaking suppresses the response (and the `UserTurnInferenceCompletedFrame` broadcast) and leaves the turn open for the next inference.
