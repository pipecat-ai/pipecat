- Enabled MoQ **client (relay) mode** for `MOQTransport`: a bot can now dial an external `moq-relay` instead of running as its own server, which is what makes MoQ deployable behind an HTTP-ingress platform (the bot's connection is outbound-only). Pass `--moq-connect <url>` to the development runner; the mode is inferred (client when `--moq-connect` is given, server otherwise) and can be forced with `--moq-serve`/`--no-moq-serve`.
  - `--moq-connect` also accepts `unix:///path/to/relay.sock` to dial a co-located (sidecar) relay over a unix socket, using a QUIC-over-unix transport. The bot↔relay hop then needs no TLS or network configuration; the relay still exposes a network endpoint for browsers, which the client is pointed at separately.
  - Added `--moq-tls-fingerprint <hex>` for dialing a self-signed relay in client mode: the runner hands the relay's certificate fingerprint to the browser as a WebTransport `serverCertificateHashes` pin. Not needed for CA-signed relays, where the browser trusts the chain normally.
