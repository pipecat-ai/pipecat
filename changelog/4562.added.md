- Added `BotRecordingSilencePadder`, a new sibling processor in `pipecat.processors.audio` that preserves wall-clock gaps in the bot recording track. Place it between `transport.output()` and `AudioBufferProcessor` and it pushes silent `OutputAudioRawFrame` chunks downstream while the bot is not speaking, so the recording stays in sync with real time when no user input audio is flowing (push-to-talk transports, server-side input gating during function/tool calls, manual muting between turns). Without this, consecutive bot utterances across such gaps were concatenated in the saved recording.
