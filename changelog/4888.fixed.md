- Fixed `UserTurnCompletionLLMServiceMixin` (used by `FilterIncompleteUserTurnStrategies`) leaving an incomplete-turn (○/◐) re-prompt timeout running when the user resumed speaking within the same turn. The timeout was only cancelled on `InterruptionFrame` (which fires on a new turn), so a user who paused and then continued could be interrupted by a stale, forced-✓ re-prompt against a partial transcript. The pending timeout is now also cancelled on `VADUserStartedSpeakingFrame`.
