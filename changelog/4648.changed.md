- `TavusTransport` now delivers bot audio via the Tavus `conversation.echo` app message API instead of a WebRTC audio track. By default, audio is sent paced to real playback time (compatible with downstream processors like `AudioBufferProcessor`). Set `TavusParams(audio_out_faster_than_realtime=True)` to instead accumulate and send audio in 100ms chunks as fast as possible, which gives Tavus more of a rendering buffer at the cost of losing realtime pacing.
  - The default `persona_id` changed from `"pipecat-stream"` to `"pipecat0"`, which signals Tavus to expect audio over the `conversation.echo` app message API instead of a WebRTC custom audio track.
