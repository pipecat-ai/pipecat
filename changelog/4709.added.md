- `FunctionSchema` now accepts an optional `handler`. When set, the LLM service registers it automatically wherever the schema is advertised in an `LLMContext` (or via an `LLMSetToolsFrame`), so no separate `register_function` call is needed. This extends the existing auto-registration of direct functions to `FunctionSchema`-based tools: the advertised tool set stays the single source of truth, so dropping a handler-carrying schema unregisters its handler too. A `FunctionSchema` without a handler stays advertise-only.
