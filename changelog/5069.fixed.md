- Fixed function calls from an interrupted generation still executing when using sequential function calls (`run_in_parallel=False`). Calls still waiting in the sequential runner queue when an interruption arrived were never cancelled and ran afterwards — a hallucinated `end_call` from a prematurely-triggered generation could hang up on a caller who was still talking. Interruptions also killed the sequential runner task itself, so function calls from all subsequent generations were silently dropped for the rest of the session. Queued calls with `cancel_on_interruption=True` are now cancelled on interruption (async calls with `cancel_on_interruption=False` stay queued), and the runner keeps serving calls from new generations.
