- Added `on_recording_started` and `on_recording_stopped` event handlers to `AudioBufferProcessor`. They fire only on actual recording state transitions, so code can react to recording being started or stopped (e.g. UI indicators, logging) without owning the processor or being the caller that triggered it — useful now that recording can be controlled from within the pipeline via `AudioBufferStartRecordingFrame` / `AudioBufferStopRecordingFrame`. `on_recording_stopped` fires after the final buffered audio has been emitted.
