- Fixed a `KeyError: 'role'` crash in `AnthropicLLMAdapter` when the context contained an empty or incomplete "thought" message. Under the interleaved-thinking beta, `claude-sonnet-5` can emit thinking blocks with no text or signature; these were stored as `LLMSpecificMessage` entries and then forwarded verbatim (as a dict with no `role`), crashing the next Anthropic serialization pass. Such thoughts are now dropped both when the context is built and during adapter conversion, so they can no longer break serialization (notably on Anthropic turns after tool calls, and when sharing a context across providers via `LLMSwitcher`).
