- Direct-function handlers now stay in sync with the tools advertised in the `LLMContext`: advertising a direct function (in `LLMContext(tools=[...])` or via an `LLMSetToolsFrame`) auto-registers its handler, and dropping it auto-unregisters the handler (an empty set, e.g. `LLMSetToolsFrame(tools=[])`, clears them all). The advertised tool set is the single source of truth, so a smaller model can't successfully call a tool you've taken away. Explicitly registered handlers, the catch-all handler, and built-in tools are never auto-pruned.
