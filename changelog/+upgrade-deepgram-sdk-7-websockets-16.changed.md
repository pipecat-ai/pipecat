- Widened the `deepgram` extra to `deepgram-sdk>=6.1.1,<8` so installations can resolve to either deepgram-sdk 6.x or 7.x. `DeepgramSTTService` now handles the `agent_rest` keyword argument that deepgram-sdk 7.2.0 added to `DeepgramClientEnvironment`, so custom `base_url` configuration keeps working on both 6.x and 7.x.
- Dropped the upper bound on the `websockets-base` extra (`websockets>=13.1`) so downstream deployments can resolve to websockets 16.x and beyond. Pipecat's `websockets` usage relies only on the modern `websockets.asyncio` API plus a handful of public symbols, all of which are retained in 16.x. Note that environments which also install the `nvidia` extra will still resolve to `websockets==15.0.1` because `nvidia-riva-client==2.25.1` pins `websockets<16`; this is unchanged behavior.
