- Fixed `ExternalUserTurnStopStrategy` triggering inference on a previous speech chunk's transcript, dropping the user's latest speech from the LLM run. `UserStoppedSpeakingFrame` is a system frame, so it is dispatched ahead of the queued final `TranscriptionFrame` it was pushed after (as with Deepgram Flux end-of-turns) and reaches the strategy first. That race alone is benign — the strategy falls back to its timer and waits for the late final — but if the user resumed speaking before the timer fired, the turn stayed open, the strategy was never reset, and the stale text satisfied the next chunk's immediate trigger. Inference then ran on the previous chunk's text while the current chunk's transcript landed after the LLM run and was aggregated into the following user turn. The strategy now clears accumulated text when the user resumes speaking, so only text finalized for the current chunk can trigger a turn stop.
