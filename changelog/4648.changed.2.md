- `TavusVideoService` now sends bot audio to Tavus via the `conversation.echo` app message API instead of a WebRTC audio track. Audio is accumulated and sent in 100ms chunks, as fast as possible.
  - The default `persona_id` changed from `"pipecat-stream"` to `"pipecat0"`, which signals Tavus to expect audio over the `conversation.echo` app message API instead of a WebRTC custom audio track.
